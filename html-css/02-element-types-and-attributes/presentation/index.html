Рядкові і блочні елементи html

Блочні елементи займають всю доступну їм ширину і висота елемента визначається його вмістом (текст або інший вкладений елемент). Такі елементи розміщуються на окремому рядку.

Рядкові елементи - це теги, які поводяться подібно до звичайного тексту і цілком можуть бути розміщені всередині блокового елемента. В основному призначені для того, щоб змінити зовнішній вигляд тексту

Основні складові тегу

  1. Тег. У елементів, які мають вміст, їх зазвичай два – відкриває та закриває.
  2. Атрибут. Атрибут містить певну інформацію щодо елемента. В одного елемента їх може бути кілька, а може й не бути.
  3. Значення атрибута.
  4. Вміст елемента.
  5. Елемент. Його межі описуються відкриваючим та закриваючим тегами.

Семантика

Тег <html> є контейнером, який містить у собі весь вміст веб-сторінки, включаючи теги <head> та <body>. Відкриває та закриває теги <html> у документі необов'язкові, але гарний стиль диктує неодмінне їх використання.
Тег <head> містить інформацію про сторінку. У ньому розташовуються метатеги, посилання на модулі, що підключаються.
Тег <body> є контейнером для всього вмісту, яке буде відображено користувачеві.

Основні правила розмітки

  1. Елементи не повинні перетинатись. Іншими словами, якщо тег, що відкриває розташовується всередині елемента, то й відповідний закриваючий тег повинен розташовуватися всередині цього елемента.
  2. Блочні елементи можуть містити текстові елементи, а також вкладені блокові.
  3. Строкові елементи можуть містити вкладені текстові елементи.
  4. Строкові елементи не можуть містити вкладені блокові елементи

Тег картинки: <img>

Тег <img> - виводить зображення на веб-сторінку у форматі jpeg (jpg), webp, svg, png та інших. Атрибутом src вказується маршрут до розташування файлу картинки для її відображення.

src – шлях до файлу зображення.

alt – альтернативний текст, що відображається у випадку коли картинка не виводиться.

<img src="" alt="" width="" height="">

Гіперпосилання

Гіперпосилання (англ. hyperlink) - Елемент на сторінці, що перенаправляє до іншого файлу. Це може бути локальний файл або розміщений у мережі Інтернет

<a href="url_path" _target="" rel="" title="">

Абсолютний шлях - https://example.ua/". Відносний шлях – вказує шлях щодо поточного файлу. Відносний шлях виглядає так: директорія(точне ім'я)/файл(з розширенням)

Позначення щодо відносного маршруту:
  • / - корінь, для сторінок це головна;
  • ./ - вказує поточну папку, де знаходиться файл;
  • ../ - піднятися на один рівень вище щодо поточної папки (вийти з папки).

Валідація HTML коду

Валідний код html - це написаний код відповідно до всіх рекомендацій специфікації консорціуму W3C. Валідація необхідна для пошукових роботів, оскільки вони працюють за стандартами і при грубих помилках розпізнавання контенту на сайті буде некоректним і вплине на видачу сторінки під час її пошуку.

https://validator.w3.org/ - сторінка для перевірки валідності html коду
