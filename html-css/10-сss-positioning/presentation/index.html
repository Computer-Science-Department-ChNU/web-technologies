position

Властивість position визначає тип позиціювання відносно вікна браузера або інших елементів (Приклади у директорії examples!):

  • static
  • relative
  • absolute
  • fixed
  • sticky
  • initial
  • inherit

fixed vs sticky

CSS position: sticky ідеально підходить, коли ви хочете, щоб елемент був тимчасово закріплений у контейнері, але при цьому залишався частиною потоку документа. Це корисно, коли ви хочете, щоб хедер, бічна панель або інший елемент із закликом до дії (Call to action - CTA) залишався видимим під час прокрутки, але не обов'язково фіксувався на всьому полі перегляду.

CSS position: fixed можна використовувати в таких випадках, коли елемент потрібно зафіксувати відносно області перегляду, незалежно від прокрутки його батьківського контейнера. Він ідеально підходить для таких елементів, як панелі навігації, які ви хочете бачити завжди, незалежно від того, як сильно користувач прокручує сторінку. І коли на елемент не має впливати на прокрутку батьківського контейнера.

z-index

Властивість z-index визначає порядок стека елемента. Іншими словами, ми можемо розмістити елемент на сторінці вище іншого елементу або перед ним.

Будь-які позиційовані елементи на веб-сторінці накладаються один на одного в певному порядку, імітуючи тим самим невидимий третій вимір, котрий перпендикулярний екрану. Кожен елемент може перебувати в цьому стеку, як нижче, так і вище інших об'єктів на веб-сторінці.

Ця властивість працює тільки для елементів, у яких значення положення задано як absolute, relative або fixed (position:absolute, position:relative, position:fixed).

overflow

Властивість overflow вказує, що зробити для переповненого елемента.
overflow: visible|hidden|scroll|auto|initial|inherit;

  • Обрізати вміст, що поза межами елемента, без можливості його перегляду.
  • Відобразити вміст, що поза межами елемента.
  • Обрізати вміст, що поза межами елемента та для можливості перегляду обрізаного тексту, відобразити смуги прокрутки.

hidden - Контент обрізається, без надання прокрутки.
scroll - вміст обрізається і браузер використовує елементи прокрутки, не має значення чи було обрізано вміст чи ні. Це уникає багатьох проблем щодо появи прокручувань і їх зникнення в динамічному середовищі. Принтери можуть друкувати переповнене вміст.
auto - смуги прокручування з'являються, тільки при необхідності

visibility (гра в хованки)

Властивість visibility визначає, чи буде елемент приховано. Властивість призначена для відображення або приховування елемента, включаючи рамку навколо нього і фон. Під час приховування елементу, хоча він і стає прихованим, місце, яке він займає, залишається за ним. Якщо передбачається вивід різних елементів в одне і те ж місце екрану, для обходу цієї особливості слід використовувати абсолютне позиціювання або скористатися властивістю display.

visibility: visible|hidden|collapse|initial|inherit;
