Pug в Node.js (2016) (раніше відомий як Jade: 2012 - 2016) - це двигун шаблонів, який використовує регістрозалежний синтаксис для створення HTML. Іншими словами, він повертає рядок HTML, відтворений згідно з даними, визначеними у файлі Pug. Можна сказати, що Pug - це посередник, який виконує завдання перетворення введених даних та перекладу їх у синтаксис HTML.

Ця зміна назви сталася через конфлікт із зареєстрованою торговою маркою "Jade" і викликану патентно-правовою домовленістю. Отже проєкт перейменували на "Pug", причому синтаксис та функціональність залишилися практично незмінними.

Node Package Manager

NPM (Node Package Manager) - це платформа та менеджер пакетів, які використовуються для розробки програм на мові програмування JavaScript і середовищі виконання Node.js. NPM дозволяє розробникам легко управляти залежностями (бібліотеками та модулями), які використовуються в їхніх проєктах.

Node Package Manager (NPM)

Дозволяє:

  • Легко встановлювати та оновлювати пакети.
  • Ділитися власними пакетами з іншими розробниками.
  • Керувати залежностями проекту.

Чому використовують NPM?

  • Велика екосистема: десятки тисяч пакетів у відкритому доступі
  • Простота встановлення: будь-який пакет можна додати однією командою.
  • Оновлення: підтримка нових версій для всіх залежностей проєкту.
  • Спільнота: спільний доступ до інструментів, розширень та бібліотек.

Ініціалізація проєкту з NPM

  • Команда для створення package.json, який зберігає інформацію про проєкт та залежності: npm init
  • Відповідайте на питання або використовуйте npm init -y для швидкого створення з налаштуваннями за замовчуванням.
  • Файл package.json містить метадані проєкту та перелік встановлених залежностей..

Встановлення пакетів

  • Встановлення локального пакету (для цього проєкту): install "package-name"
  • Встановлення локального пакету з версією (для цього проєкту): install "package-name@versiom"
  • Встановлення глобального пакету (доступного в системі): npm install -g "package-name"

Основні команди NPM

  1. Встановлення пакету: npm install "назва_пакету"
  2. Оновлення пакету: npm update "назва_пакету"
  3. Видалення пакету: npm uninstall "назва_пакету"
  4. Запуск скриптів: npm run "назва_скрипту"
  5. Список глобальних пакетів: npm list -g
  6. Очищення кешу: npm cache clean

Залежності та package-lock.json

  1. package.json: основний файл конфігурації NPM.
  2. package-lock.json: автоматично створений файл, який містить точні версії всіх встановлених залежностей.
  3. "dependencies": потрібні для роботи програми
  4. "devDependencies": потрібні тільки для розробки (команда npm install --save-dev).

Semantic Versioning

У Node Package Manager (NPM) версії записуються за стандартом Semantic Versioning, і версії, такі як "^5.1.0", вказують певний діапазон сумісності. Основні частини семантичного версіонування. Формат версій виглядає як MAJOR.MINOR.PATCH:

  • MAJOR — головна версія: змінюється, коли відбуваються не зворотно сумісні зміни.
  • MINOR — мінорна версія: змінюється, коли додається новий функціонал, але зберігається зворотна сумісність.
  • PATCH — патч: змінюється, коли виправляються помилки без додавання нових функцій або порушення сумісності.
About Semantic Versioning

Semantic Versioning

Значення символів у версіях

  • ^: ця позначка означає, що можна використовувати будь-яку версію, де головна версія (MAJOR) не змінюється, але дозволені зміни в MINOR і PATCH. Наприклад, "^5.1.0" означає, що NPM встановить будь-яку версію, починаючи від 5.1.0 і аж до 6.0.0 (але не включаючи 6.0.0), тобто 5.x.x.
  • ~: символ ~ дозволяє оновлювати лише PATCH-версії, зберігаючи конкретну MINOR-версію. Наприклад, "~5.1.0" дозволить оновлення від 5.1.0 до 5.1.x (де x може бути будь-яким), але не більше.
  • Без символів (5.1.0): вказує на чітку конкретну версію, і NPM буде використовувати тільки її, без оновлення.

Semantic Versioning

Приклади

  • "^5.1.0" ➔ дозволяє версії від 5.1.0 до <6.0.0
  • "~5.1.0" ➔ дозволяє версії від 5.1.0 до <5.2.0
  • "5.1.0" ➔ використовується тільки версія 5.1.0

Встановлення Gulp

Gulp (Gulpjs Quick Start ) - це інструмент для автоматизації рутинних задач у веброзробці. Він допомагає спростити процес роботи з вебпроєктами, забезпечуючи автоматичне виконання завдань, таких як компіляція із препроцесорів (наприклад, Sass чи LESS), оптимізація зображень, об'єднання та мініфікація файлів JavaScript і CSS, попереднє відстеження змін в файлах тощо.

Основна ідея Gulp полягає у використанні концепції «завдань» (tasks) і «плагінів» (plugins).
Завдання - це прості функції, які виконують певні операції.
Плагіни - це зовнішні розширення, які використовуються для конкретних завдань.
Gulp дозволяє вам створювати ланцюжки завдань, які будуть виконуватися послідовно при запуску команди.

Основні переваги Gulp

  • Автоматизація задач: Gulp дозволяє автоматизувати багато рутинних задач, що допомагає зекономити час розробників і знизити ризик виникнення помилок.
  • Легкість використання: Створення та налаштування завдань у Gulp відбувається за допомогою коду на JavaScript. Це дозволяє розробникам більш гнучко налаштовувати свої процеси автоматизації.
  • Багатофункціональність: Завдяки великій кількості плагінів, Gulp може виконувати різноманітні завдання: від оптимізації зображень до обробки CSS і HTML.
  • Чистий код: Gulp дозволяє створювати покращені робочі потоки, які полегшують обслуговування коду і покращують загальну якість проєкту.

Основи Pug: синтаксис, теги, текст

Коли мова йде про базовий синтаксис Pug, центральна роль відведена індентації (відступам) та ефективності. На відміну від HTML, де вкладеність реалізується за допомогою відкриваючих та закриваючих тегів, в Pug це здійснюється за допомогою індентації — це коли вкладені елементи розташовані праворуч від батьківських, створюючи зрозумілу структуру на подобі дерева. Цей принцип часто порівнюють із мовою Python, де відступи та індентація також відіграють ключову роль в блоках коду.

Завдяки цьому вам не потрібно турбуватися про закриваючі теги, які можуть «розгубитися» в купі коду, особливо в великих проєктах.

Крім того, виразність індентації дозволяє швидко ідентифікувати глибину вкладеності елементів, що може бути край корисним при швидкому ревью коду або його рефакторингу.

Що таке SASS?

  • SASS (Syntactically Awesome Style Sheets) – мова стилізації, що компілюється в CSS.
  • Забезпечує покращену функціональність та можливості, які відсутні в звичайному CSS.
  • Популярний у веб-розробці через можливість писати гнучкий, модульний код.

Основні Особливості SASS

  • Змінні (Variables) – для зберігання значень, які використовуються повторно.
  • Вкладеність (Nesting) – дозволяє писати вкладені стилі, схожі на HTML-структуру.
  • Міксини (Mixins) – набір стилів, які можна повторно використовувати з параметрами.
  • Успадкування (Inheritance) – дозволяє ділити стилі між різними класами.
  • Функції та Обчислення – можливість виконувати обчислення, працювати з кольорами та розмірами.

Встановлення та Налаштування SASS

  • Інсталяція через npm: npm install -g sass.

Компіляція SASS у CSS

  • Використання команди в терміналі: sass style.scss style.css
  • Альтернативні способи: автоматичні компілятори, такі як Gulp, Vite, Webpack.

Практичне застосування

  • Розбивка на модулі: структура файлів у SASS.
  • Рекомендовані директорії:
    • _variables.scss
    • _mixins.scss
    • _base.scss
    • _layout.scss
    • _components.scss

Переваги та Недоліки SASS

  • Переваги:
    • Зручність і гнучкість у написанні стилів
    • Читаємість коду та його модульність
    • Швидша розробка та підтримка проектів.
  • Недоліки:
    • Потребує компіляції у CSS
    • Може стати важким для підтримки при надмірному вкладенні.ь
